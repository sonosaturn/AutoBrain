## 💡 Proposal
Based on the project context, particularly the high latency in vector search (**690ms**) and the existing logic for tracking **wikilink frequencies** and **ghost links**, I propose the following technical improvement:

### **Proposal: Graph-Augmented RAG (Re-ranking via Link Centrality)**

**The Problem:** 
The current retrieval process (`get_context_for_query`) relies heavily on vector search, which is slow and potentially lacks structural awareness. While the system identifies important "hubs" via `get_wikilink_frequencies`, this structural data is currently used for maintenance (like removing ghost links) rather than improving the user’s chat experience.

**The Solution:**
Modify `autobrain_core/graph_manager.py:get_context_for_query` to implement a **Graph-Weighted Re-ranker**. 

Instead of returning the raw top-K results from the vector/BM25 search, the system should adjust the relevance score of a document based on its "Centrality" within the Autobrain.

**Technical Implementation:**
1.  **Pre-compute Centrality:** Leverage the existing `get_wikilink_frequencies` logic to generate a "Importance Score" for every node in the graph. Nodes with higher incoming wikilinks are treated as "Source of Truth" or "Hub" documents.
2.  **Hybrid Scoring:** In the `get_context_for_query` function, calculate the final relevance score using:
    *   `Final_Score = (Vector_Similarity * 0.7) + (Normalized_Link_Frequency * 0.3)`
3.  **Context Expansion:** If a retrieved document has a very high `wikilink_frequency`, automatically include its "Correlati" (Related) section (validated by `_rule_has_correlati_section`) into the context window, even if those specific related notes didn't hit the vector threshold.

**Why this is useful:**
*   **Accuracy:** It prioritizes well-established, highly-connected notes over obscure or isolated snippets that might coincidentally match a vector query.
*   **Efficiency:** By using graph density to prune irrelevant results, you can potentially reduce the `top_k` required from the expensive vector search, mitigating the 690ms bottleneck.
*   **Consistency:** It aligns the AI’s responses with the "Autobrain" structure the user has built, ensuring that the most "important" concepts (as defined by their link frequency) are given priority in `build_system_prompt`.

**Impact on Process:**
This improvement directly links **proc_24_main** (frequency analysis) with **proc_12_chat** (context retrieval), turning a maintenance metric into a retrieval feature.
