## 💡 Proposal
As the Creative Leader, I propose shifting from a **reactive error state** to a **proactive self-healing environment.**

### The Proposal: "GitNexus Doctor" Diagnostic Wrapper

Instead of allowing a raw `[WinError 2]` (which is a cryptic Windows OS error) to reach the user, we should implement a **Dependency Auto-Discovery & Health Check** module.

#### The Problem:
`WinError 2` occurs because the Git executable (or a specific script) is missing from the system's `PATH`. This is a friction point that stalls developer productivity and triggers support tickets.

#### The Technical Improvement:
Implement a **Pre-Flight Path Validator** using a "Search & Suggest" pattern.

1.  **Intercept the Exception:** Wrap the query execution in a specialized handler that specifically looks for `FileNotFoundError` (WinError 2).
2.  **Automated Search:** If the error triggers, the system shouldn't just quit. It should immediately scan common installation directories (e.g., `C:\Program Files\Git\bin`, `C:\Users\<User>\AppData\Local\Programs\Git`) to locate the missing binary.
3.  **Proactive Resolution:**
    *   **Level 1 (Automatic):** If found, the system temporarily appends the path to the current process environment and retries the query seamlessly.
    *   **Level 2 (Guided):** If not found, instead of a raw error, display a **"GitNexus Doctor" CLI prompt**:
        > "⚠️ Git executable not found in your PATH. We looked in [X, Y, Z] but couldn't find it. Please provide the path to `git.exe` or run `nexus setup` to reconfigure."

#### Why this is the priority:
*   **User Experience:** It transforms a "broken" experience into a "guided" one.
*   **Support Reduction:** It eliminates the most common "Environment Variable" troubleshooting sessions.
*   **Resilience:** It makes GitNexus robust against system updates or environment changes that might strip the PATH variable.

**Implementation Note:** This can be achieved in Python by checking `shutil.which("git")` during initialization and providing a custom fallback mechanism if it returns `None`.
