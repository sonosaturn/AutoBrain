## 💡 Proposal
Based on the project context, particularly the interaction between the chat process (`proc_16`), the wikilink frequency analysis (`proc_22`), and the response logging (`proc_17`), I propose the following technical improvement:

### **Improvement: Semantic Wikilink Auto-Injection Layer**

**The Problem:**
The project relies heavily on a "wikilink" structure for its knowledge graph (as seen in `get_wikilink_frequencies` and `_rule_has_wikilinks`). However, currently, the chat process (`proc_16`) and the logging/summarization process (`proc_17`) appear to treat AI responses as passive text. If the AI fails to manually format a term as a wikilink, that information remains isolated from the graph, leading to the "ghost links" or "isolated wikilinks" that `remove_ghost_links.py` tries to clean up later.

**The Solution:**
Integrate a **Semantic Wikilink Auto-Injection Layer** within the `summarize_ai_response` function (in `autobrain_core/logger.py`). 

Instead of just summarizing the response for logs, this layer will:
1.  **Cross-Reference:** Scan the AI's response against the high-frequency wikilinks identified by `get_wikilink_frequencies`.
2.  **Auto-Link:** Automatically wrap keywords in `[[wikilink]]` syntax if they exist in the graph but weren't linked by the AI.
3.  **Validate:** Run the `validator.py` rules (like `_rule_no_cot_markers` and `_rule_has_h1_title`) in-stream to sanitize the output before it is committed to the long-term memory or the UI.

**Technical Implementation:**
*   **Module:** Modify `autobrain_core/logger.py`.
*   **Dependency:** Import `get_wikilink_frequencies` from `materialize_nodes.py`.
*   **Logic:** 
    ```python
    # Pseudo-logic addition to summarize_ai_response
    known_entities = get_wikilink_frequencies()
    for entity in known_entities:
        if entity in raw_response and not is_already_linked(entity):
            raw_response = raw_response.replace(entity, f"[[{entity}]]")
    ```

**Why this is useful:**
1.  **Graph Integrity:** It ensures the knowledge graph grows organically with every chat interaction without relying on the AI's "hallucinated" formatting capabilities.
2.  **Performance Optimization:** By reinforcing the link structure, the `get_context_for_query` function can rely more on fast BM25/keyword lookups (currently only 14ms) and rely less on the expensive Vector search (currently 616ms).
3.  **Consistency:** It enforces the project's architectural rules (from `validator.py`) programmatically at the edge of the interaction.

**Creative Leader Note:**
This transforms the logging process from a "black box" record-keeper into an active **Knowledge Materializer**, ensuring that every interaction strengthens the "Autobrain" without requiring more complex prompting.
